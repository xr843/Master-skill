# Desktop Manager v0.10.0 发布计划(基线变绿 → 发布 → 观测)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 desktop Quality Gate 由 Unproven 变绿(fidelity dry-run 基线),给 v0.10.0 发布配齐三平台二进制 + README 桌面端章节,并设置 7 月 24 日采用度观测点。

**Architecture:** 三条线:(A) 给 `master-skill-desktop` 加 `--baseline` headless 子命令,复用 GUI 同一 CLI 包装与 TraceStore 持久化,使基线可脚本化;(B) 新增 `release-desktop.yml` GitHub Actions 矩阵构建,在 release published 时把 Linux/Windows/macOS 二进制附到 release;(C) README(中英)加桌面端章节 + 绿门禁截图,按 `.version-bump.json` 约定 bump 到 0.10.0,打 tag 发布(复用现有 npm-publish OIDC 流程)。

**Tech Stack:** Rust (egui/eframe 0.31), GitHub Actions, Python fidelity runner (`scripts/test-fidelity.py`)。

## Global Constraints

- 仓库工作流:所有变更走分支 + PR 合并到 `main`;commit 前缀沿用 `feat(desktop):` / `ci:` / `docs:` / `chore(release):` 约定。
- fidelity 基线一律用 `--dry-run`(零 API 费用);不要触发付费 LLM eval。
- 版本号 0.10.0 必须同步 `.version-bump.json` 列出的全部 5 个文件,并通过 `python3 scripts/check-manifest-versions.py` 校验。
- `desktop/Cargo.toml` 的 crate 版本独立(0.1.0),本次不动。
- 不新增 GUI 面板/功能;Task 1 是唯一的代码改动且为 headless 插件。
- 用户本地仓库 `~/projects/master-skill` 有未提交改动(`hooks/run-hook.cmd`),不得触碰或还原。
- README 中文为主文档,README_EN 同步英文;截图放 `docs/assets/`。

### Task 2: `release-desktop.yml` 三平台二进制工作流(worktree B,可与 Task 1 并行)

**Files:**
- Create: `.github/workflows/release-desktop.yml`

**Interfaces:**
- Consumes: 仓库现有 workflow 约定(`npm-publish.yml`:`on: release: types: [published]` + `workflow_dispatch`;所有 `uses:` 钉全长 commit SHA + 版本注释,照抄现有 workflow 里已钉过的 checkout/setup action SHA)。
- Produces: release published 时自动把 `master-skill-desktop-{linux-x86_64,windows-x86_64.exe,macos-aarch64}` 附到该 release;`workflow_dispatch` 时只构建并上传 artifacts(不碰 release)。

- [ ] **Step 1: 写 workflow**,要点:
  - `on: release: types: [published]` + `workflow_dispatch`;`permissions: contents: write`(附 release 资产所需,workflow 顶部注释说明)。
  - matrix:`ubuntu-latest`(产物 `master-skill-desktop-linux-x86_64`)、`windows-latest`(`master-skill-desktop-windows-x86_64.exe`)、`macos-latest`(`master-skill-desktop-macos-aarch64`)。
  - Linux job 先 `sudo apt-get update && sudo apt-get install -y libxkbcommon-dev libwayland-dev libxcb-render0-dev libxcb-shape0-dev libxcb-xfixes0-dev`(eframe/winit 构建依赖)。
  - `dtolnay/rust-toolchain@stable`(钉 SHA)或直接用 runner 自带 stable;`cargo build --release --manifest-path desktop/Cargo.toml`。
  - 重命名产物 → release 事件时 `gh release upload "$GITHUB_REF_NAME" <file>`(env `GH_TOKEN: ${{ github.token }}`);dispatch 时 `actions/upload-artifact`(钉 SHA)。
- [ ] **Step 2: 本地静态校验**:有 `actionlint` 就跑;没有则 `python3 -c "import yaml,sys; yaml.safe_load(open('.github/workflows/release-desktop.yml'))"`,预期无错误。
- [ ] **Step 3: Commit** `ci: build desktop release binaries` 并开 PR。
- [ ] **Step 4: PR 合并后(Task 5 之前可先行)**:`gh workflow run release-desktop.yml --repo xr843/Master-skill`,等 3 个 job 全绿、artifacts 齐全,作为发布前演练。失败则修复后重跑。
