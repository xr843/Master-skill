# 教学模式选择决策树

> **何时读这个**：用户犹豫该用 `/compare-masters` / `/master-debate` / `/master-curriculum` 哪个时；用户问"哪些祖师对这个问题怎么看"但意图不明时。

> **这棵树现在有可执行版本。** 判定顺序与关键词已固化到仓库根的 `routing.json`，
> 由 `master-skill recommend "<用户原话>"`（确定性）和 `/master-help`（聊天内）共用。
> 本文下方"示例输入分类"的 10 条即 `tests/cli.test.mjs` 的回归 fixture——
> **改这里的表就会让测试失败**，这是刻意的：散文与实现不允许各走各的。

## 三模式速查

| 模式 | 维度 | 轮次 | 输出 | 适合场景 |
|------|------|------|------|---------|
| `/compare-masters` | 横向 / 并列 | 单轮 | 多位祖师对同一问题的并列答案 | 想看不同传统 / 宗派的视角差异，**不需要互相反驳** |
| `/master-debate` | 横向 / 对立 | 4 轮 | 立论 → 反驳 → 回应 → 综合 + 教内余争 | 想看争议议题的**深入分歧**，关心"谁的见地更究竟" |
| `/master-curriculum` | 纵向 / 时序 | 单次规划 | 「根基 → 深入 → 精研 → 盲点」学修路径 | **不是比较，是请求学习计划** |

## 决策树（用户表达 → 模式）

```
用户问"X 问题"时：
│
├─ 包含"学习计划 / 路径 / 入门 / 先学什么 / 下一步读什么 / curriculum"
│     → /master-curriculum
│
├─ 包含"辩论 / 各执一词 / 谁更对 / 究竟 / 高下 / 争议 / debate"
│     → /master-debate
│
├─ 包含"对比 / 比较 / 不同视角 / 各派怎么看 / X vs Y / compare"
│     │
│     ├─ 议题本身有传统对立性（禅净 / 空有 / 顿渐 / 应成 vs 顿悟）
│     │     │
│     │     ├─ 用户语气中性 / 想看多角度 → /compare-masters
│     │     └─ 用户想看深入分歧 / 谁更对 → /master-debate
│     │
│     └─ 议题中性（如"慈悲 / 业 / 缘起"）
│           → /compare-masters
│
└─ 没有比较 / 辩论 / 学习计划意图，只是问"X 怎么修"
      → 单 master skill（按传承推荐：禅 → huineng / 净土 → yinguang / 中观 → kumarajiva / 道次第 → atisha 或 tsongkhapa / 内观 → mahasi-sayadaw 等）
```

## 关键关键词速查

### `/compare-masters` 触发关键词

- 对比 / 比较 / compare / 不同视角 / 各派怎么看 / 各位祖师怎么看 / 多个角度
- 显式举出 2-N 位祖师并问"他们怎么说" → 平铺单轮答

### `/master-debate` 触发关键词

- 辩论 / 各执一词 / 谁更对 / 究竟 / 高下 / 争议 / debate / 顿渐之争 / 性相之辩 / 空有之争 / 禅净之争
- 问"X 派和 Y 派的核心分歧" → 4 轮对辩
- 关键：用户想看**对抗式反复推敲**，不是平铺

### `/master-curriculum` 触发关键词

- 学修次第 / 学习计划 / 入门 / 先学什么 / 从哪开始 / 开始学 / 应该读 / 路径推荐 / 按什么顺序 / curriculum / roadmap
- 问"禅宗从哪开始学" → 时序路径，不是比较

> ⚠️ `怎么修` **不是**本模式的触发词。它是求法，不是求路径——"这个怎么修"归单 master skill（见本文决策树末端），"从哪开始学"才归这里。本文此处曾同时把 `怎么修` 列为 curriculum 触发词又在决策树里判给单 master，两处矛盾已按后者裁定。

## 示例输入分类

| 用户原话 | 模式 | 理由 |
|---------|------|------|
| "禅宗和净土宗怎么比较" | `/compare-masters` | 平铺对比，无对立请求 |
| "禅净之争究竟谁更对" | `/master-debate` | 显式对立 + 问"谁更对" |
| "我想学禅宗从哪开始" | `/master-curriculum` | 个人学习路径 |
| "天台和华严的圆教有什么不同" | `/compare-masters` | 对比但无对抗语气 |
| "性相之辩玄奘和鸠摩罗什会怎么辩" | `/master-debate` | 议题对立 + 显式辩论框架 |
| "刚开始学藏传，应该读什么" | `/master-curriculum` | 入门路径请求 |
| "应成中观和顿悟禅的分判" | `/master-debate` | 涉及"分判" → 深入分歧 |
| "印光大师和虚云老和尚的修法异同" | `/compare-masters` | "异同" → 平铺，非对辩 |
| "中观、唯识、华严的判教高下" | `/master-debate` | "高下" → 对辩 |
| "想综合学习汉传佛教，应该按什么顺序" | `/master-curriculum` | "按什么顺序" → 时序 |

## 选错模式时的兜底

用户在某模式中明显错位时（如 `/compare-masters` 输入"谁更对"），优雅建议切换：

> 「您似乎想看深入的对辩交锋，建议改用 `/master-debate`，4 轮立论-反驳-回应-综合更适合您的问题。是否切换？」

## 单 master skill vs 模式选择

如果用户的问题**没有任何比较 / 辩论 / 计划意图**，只是想知道"X 议题怎么修"或"X 祖师怎么说"：

- 单一传统议题 → 直接用对应 master skill（如 `/master-yinguang` 答念佛、`/master-huineng` 答见性）
- 不知道哪位祖师 → 按 `references/traditions.md` §"选择传统的常见判断" 表推荐入门

> 模式 skill（compare / debate / curriculum）成本更高（需聚合多 master 数据）。如非必要，先用单 master skill。
